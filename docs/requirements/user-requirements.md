# 用户需求文档 - 聚集搜索

## 产品概述

**产品名称**: 聚集搜索
**产品定位**: 综合搜索应用（类似深度聚集搜索）
**目标用户**: Deepin 25 用户，需要快速搜索应用、文件，二期扩展到联网搜索

---

## 核心需求

### 1. 用户场景

> 用户需要在一个应用中快速查找和启动已安装的应用程序，搜索本地文件，并浏览所有应用。
>
> 搜索应该支持中文和拼音，结果应该按类型分组显示，方便用户快速找到目标。

### 2. 用户价值

- **效率提升**: 无需在多个界面切换，一个应用完成所有搜索
- **智能搜索**: 支持拼音和中文，降低输入成本
- **清晰展示**: 分类分组，结果一目了然

## 功能需求（一期）

### 主界面

#### 左侧导航栏
| 功能点 | 说明 |
|--------|------|
| 搜索入口 | 主搜索页面，支持应用和文件搜索 |
| 应用浏览 | 以网格形式展示所有已安装应用 |
| 文件浏览 | 浏览所有索引的文件（按类型分组） |
| 近期使用 | 显示最近启动的应用和文件（二期） |
| 收藏夹 | 用户收藏的应用和文件（二期） |
| 设置 | 应用设置界面 |
| 用户中心 | 用户账户中心（二期） |
| 索引控制 | 开始/停止文件索引 |

#### 搜索界面
| 功能点 | 说明 |
|--------|------|
| 搜索框 | 居中显示，支持多行 |
| 占位符 | "搜索应用、文件或输入关键词..." |
| 清除按钮 | 输入内容后显示"×"清除 |
| 标签页过滤器 | 全部 / 应用 / 文件 / 联网（二期） |

#### 结果展示
| 功能点 | 说明 |
|--------|------|
| 分组显示 | 应用为一组，文件按类型（文档/图片/视频/音乐）分组 |
| 应用结果 | 网格卡片显示，包含图标、中文名、英文名 |
| 文件结果 | 列表显示，包含图标、文件名、路径、大小、修改时间 |
| 空状态 | 无结果时显示友好提示 |

#### 欢迎界面
| 功能点 | 说明 |
|--------|------|
| 欢迎标题 | "欢迎使用聚集搜索" |
| 快速启动卡片 | 浏览应用、浏览文件、开始搜索 |
| 搜索提示 | 简要说明搜索功能（拼音支持、筛选等） |

---

### 交互需求

#### 基础交互
| 操作 | 行为 |
|------|------|
| 点击搜索框 | 聚焦，准备输入 |
| 输入搜索词 | 实时搜索（防抖 300ms） |
| 点击清除按钮 | 清空搜索内容，聚焦搜索框 |
| 按 Esc 键 | 清空搜索内容 |
| 按 Enter 键 | 打开选中项或第一个结果 |

#### 导航交互
| 操作 | 行为 |
|------|------|
| 点击左侧导航 | 切换到对应页面 |
| 点击过滤器标签 | 过滤结果类型 |
| 点击应用卡片 | 启动应用 |
| 点击文件项 | 用默认应用打开文件 |

#### 键盘快捷键
| 快捷键 | 功能 |
|--------|------|
| Ctrl+F | 聚焦搜索框 |
| ↑↓ | 选择结果 |
| Enter | 打开选中项 |
| Esc | 清空搜索 |

## 搜索功能

### 应用搜索
- 支持中文搜索
- 支持拼音首字母搜索（如 "wx" → "微信"）
- 支持拼音全拼搜索（如 "weixin" → "微信"）
- 按应用名称模糊匹配

### 文件搜索
- 按文件名搜索
- 按文件类型分组
- 显示文件路径、大小、修改时间
- 支持常用文件格式（doc, docx, pdf, txt, jpg, png, mp4, mp3 等）

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 应用数量过多导致加载慢 | 高 | 缓存应用列表，分页展示 |
| 文件索引时间过长 | 高 | 后台索引，显示详细进度 |
| 搜索结果过多导致卡顿 | 中 | 限制结果数量，懒加载 |
| 高分辨率适配问题 | 中 | 使用响应式布局 |
