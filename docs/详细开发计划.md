# Launcher 详细开发计划

**项目名称**: Launcher - 聚焦搜索启动器
**开发策略**: 前后端分离 + 模拟数据验证
**目标平台**: Deepin 25 / Qt 5 兼容

---

## 🎯 开发策略概述

### 前后端分离模式

| 层级 | 技术 | 职责 |
|------|------|------|
| **前端 (UI 层)** | QML | 交互、动画、视觉效果 |
| **后端 (逻辑层)** | C++ | 数据处理、业务逻辑 |
| **数据层** | SQLite | 数据存储 |

## 🔄 前后端对接方式

### 数据流向
```
QML (前端)
  ↓ SearchViewModel::setQuery()
  ↓
C++ SearchEngine (后端)
  ↓ search()
  ↓
  ↓
  ↓ 返回结果 → SearchModel (QML)
  ↓
  ↓
ListView 显示
```

### QML 调用 C++
```qml
// 前端
TextField {
    id: searchInput
    onTextChanged: viewModel.query = text
}

// ViewModel 连接
SearchViewModel {
    id: viewModel
    SearchEngine* engine: SearchEngine {}
    query: ""  // → 触发搜索
}

// 后端
SearchEngine {
    onQueryChanged: {
        m_appEngine->search(query)
        m_fileRepo->search(query)
    }
}
```

### 数据序列化
```cpp
// C++ → QML
QVariantMap map;
map["type"] = result.type;
map["id"] = result.id;
map["title"] = result.title;
map["subtitle"] = result.subtitle;
map["icon"] = result.icon;
map["details"] = result.details;
```

## 🎯 开发原则

1. **渐进式验证**: 每个阶段都有明确的验收节点
2. **前后端分离**: 可以独立开发和测试
3. **模拟数据优先**: 先用模拟数据验证 UI，再对接真实数据
4. **快速迭代**: 每个节点都可以调整方向
5. **用户参与**: 每个节点都需要用户确认
